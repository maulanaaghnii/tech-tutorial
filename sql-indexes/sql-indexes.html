SQL Indexes

Introduction

Index dalam SQL berfungsi sebagai mekanisme untuk mempercepat pencarian data di dalam tabel. Ia bekerja dengan menyimpan referensi yang menunjuk langsung ke lokasi data, sehingga sistem bisa menemukan informasi yang dicari tanpa harus memindai seluruh tabel. Walaupun penggunaan index dapat meningkatkan kecepatan dalam menjalankan query SELECT, keberadaannya justru bisa memperlambat proses seperti INSERT atau UPDATE karena index juga harus diperbarui setiap kali ada perubahan data. Meskipun begitu, index tidak mengubah isi data apa pun. Secara teknis, index membutuhkan ruang penyimpanan tersendiri di database.

Namun, karena fungsinya hanya sebagai alat bantu performa, index tidak terlihat secara langsung oleh pengguna.


Index dalam SQL dapat dibuat menggunakan pernyataan CREATE INDEX. Pernyataan ini memungkinkan Anda untuk memberi nama index, menentukan tabel dan kolom mana yang akan diindeks, serta menunjukkan apakah index dalam urutan menaik atau menurun.


CREATE INDEX index_name ON table_name;

Index unik tidak hanya digunakan untuk performa, tetapi juga untuk integritas data. Index unik tidak mengizinkan nilai duplikat untuk dimasukkan ke dalam tabel. Index ini secara otomatis dibuat oleh konstrain PRIMARY dan UNIQUE ketika diterapkan pada tabel database, untuk mencegah pengguna memasukkan nilai duplikat ke dalam kolom tabel yang diindeks.


CREATE UNIQUE INDEX index_name
on table_name (column_name);

Index kolom tunggal dibuat hanya pada satu kolom tabel. Berikut adalah sintaksnya.


CREATE INDEX index_name
ON table_name (column_name);

Index komposit adalah index yang dapat dibuat pada dua atau lebih kolom dari sebuah tabel. Berikut adalah sintaks dasarnya.


CREATE INDEX index_name
on table_name (column1, column2);

Index implisit adalah index yang secara otomatis dibuat oleh server database ketika sebuah objek dibuat. Sebagai contoh, index secara otomatis dibuat ketika konstrain primary key dan unique dibuat pada tabel dalam database MySQL.


Sebuah index dapat dihapus menggunakan perintah SQL DROP. Menghapus index dapat mempengaruhi performa query dalam database. Oleh karena itu, index perlu dihapus hanya ketika benar-benar diperlukan.


DROP INDEX index_name;
